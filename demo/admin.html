Problemi Noti — Stato Attuale
🔐
Google OAuth — redirect URL non configurato
Aggiungere https://demo.poilove.com in Supabase Auth → URL Configuration → Allowed Redirect URLs. Senza questo, nessun utente riesce a registrarsi.
APERTO
👥
Profilo hardcoded — dati falsi visibili
Mirela K., Arben S., Liri V., "87 love", "Amatore" sono HTML statico in demo/index.html (righe 1686–1860). Non vengono dal database.
APERTO
🗄️
INSERT policy mancante su profiles
La tabella profiles ha SELECT e UPDATE ma non INSERT per utenti autenticati. Se il trigger handle_new_user fallisce, l'upsert JS fallisce silenziosamente e i POI non si salvano (FK violation).
APERTO
📝
Description limit: DB 200 chars vs form 1000
Il CHECK constraint in pois dice max 200 chars. Il form mostra "0 / 1000". Qualsiasi testo > 200 chars fa fallire l'INSERT senza errore visibile.
APERTO
🏠
Luoghi Personali (Casa/Lavoro) — solo localStorage
Casa e Lavoro sono salvati solo nel browser locale. Si perdono pulendo i cookie. Non vanno mai su Supabase.
DA FARE

media.poilove.com — 503 + DNS server-side rotto
Il server non risolve supabase.co. Fix: SSH → aggiungere nameserver 8.8.8.8 in /etc/resolv.conf. Upload foto usa il fallback Supabase Storage ma potrebbe non funzionare.
